# Руководство по интеграции с tmux

Полное руководство по использованию интеграции tmux в `oh-my-opencode-slim`, чтобы наблюдать за работой агентов в реальном времени через автоматическое создание pane.

## Обзор

**Наблюдайте за работой агентов в реальном времени.** Когда Orchestrator запускает подагентов или инициирует фоновые задачи, новые tmux-pane создаются автоматически и показывают живой прогресс каждого агента.

### Ключевые преимущества

- **Видимость в реальном времени** активности агентов
- **Автоматическое управление pane** - pane создаются и раскладываются автоматически
- **Интерактивная отладка** - можно подключиться к любой сессии агента
- **Мониторинг фоновых задач** - виден прогресс долгих операций
- **Поддержка нескольких сессий** - разные проекты могут иметь отдельные tmux-окружения

> **Временный обходной путь:** запускайте OpenCode с `--port`, чтобы включить интеграцию tmux. Порт должен совпадать с переменной окружения `OPENCODE_PORT` (по умолчанию: 4096). Это необходимо до исправления upstream-проблемы. [opencode#9099](https://github.com/anomalyco/opencode/issues/9099).

## Быстрая настройка

### Шаг 1: включите интеграцию tmux

Отредактируйте `~/.config/opencode/oh-my-opencode-slim.json` (или `.jsonc`):

```json
{
  "tmux": {
    "enabled": true,
    "layout": "main-vertical",
    "main_pane_size": 60
  }
}
```

### Шаг 2: запустите OpenCode внутри tmux

```bash
# Запуск новой tmux-сессии
tmux

# Запуск OpenCode с портом по умолчанию (4096)
opencode --port 4096
```

Готово. Теперь pane агентов будут создаваться автоматически.

## Конфигурация

### Настройки tmux

Настройте поведение tmux в `~/.config/opencode/oh-my-opencode-slim.json` (или `.jsonc`):

```json
{
  "tmux": {
    "enabled": true,
    "layout": "main-vertical",
    "main_pane_size": 60
  }
}
```

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `enabled` | boolean | `false` | Включить/выключить создание tmux-pane |
| `layout` | string | `"main-vertical"` | Layout-пресет (см. [Варианты layout](#варианты-layout)) |
| `main_pane_size` | number | `60` | Размер главной pane в процентах (20-80) |

### Варианты layout

Выберите, как раскладывать pane:

| Layout | Описание |
|--------|----------|
| `main-vertical` | Ваша сессия слева (60%), агенты справа столбцом |
| `main-horizontal` | Ваша сессия сверху (60%), агенты ниже |
| `tiled` | Все pane одинакового размера в сетке |
| `even-horizontal` | Все pane рядом по горизонтали |
| `even-vertical` | Все pane столбцом по вертикали |

**Пример: horizontal layout для широких экранов**
```json
{
  "tmux": {
    "enabled": true,
    "layout": "main-horizontal",
    "main_pane_size": 50
  }
}
```

**Пример: tiled layout для максимального параллелизма**
```json
{
  "tmux": {
    "enabled": true,
    "layout": "tiled",
    "main_pane_size": 50
  }
}
```

## Примеры использования

### Базовый сценарий

1. **Запустите tmux и OpenCode:**
   ```bash
   tmux
   opencode --port 4096
   ```

2. **Попросите Orchestrator делегировать работу:**
   ```
   Please analyze this codebase and create a documentation structure.
   ```

3. **Наблюдайте за автоматическим созданием pane:**
   - основная сессия OpenCode остается видимой
   - появляются новые pane с активностью агентов
   - можно переключаться между pane и следить за прогрессом

### Навигация по pane

```bash
# Показать все pane (в другом терминале)
tmux list-panes -a

# Переключение на конкретную pane (в tmux)
Ctrl+B Arrow Keys    # Перемещение между pane
Ctrl+B %             # Разделить pane по горизонтали
Ctrl+B "             # Разделить pane по вертикали
Ctrl+B z             # Увеличить/уменьшить pane
Ctrl+B c             # Создать новое окно
Ctrl+B n/p           # Следующее/предыдущее окно
```

### Отсоединение и повторное подключение

```bash
# Отсоединиться от tmux-сессии (оставить ее работать)
Ctrl+B d

# Подключиться позже
tmux attach

# Или указать конкретную сессию
tmux attach -t 0
```

### Мониторинг нескольких проектов

Для разных проектов на разных портах:

```bash
# Проект 1
tmux new -s project1
opencode --port 4096

# Отсоединиться и создать проект 2
Ctrl+B d
tmux new -s project2
opencode --port 4097

# Переключение между проектами
tmux switch -t project1
tmux switch -t project2
```

### Навигация по сессиям

```bash
# Список всех сессий (в другом терминале)
tmux list-sessions

# Переключение между сессиями (в tmux)
Ctrl+B s      # список сессий и выбор
Ctrl+B (      # предыдущая сессия
Ctrl+B )      # следующая сессия
Ctrl+B $      # переименовать текущую сессию
Ctrl+B d      # отсоединиться от текущей сессии
```

## Устранение неполадок

### Интеграция tmux не работает

**Проблема:** pane не создаются

**Решения:**
1. **Проверьте, что интеграция tmux включена:**
   ```bash
   cat ~/.config/opencode/oh-my-opencode-slim.json | grep tmux # (или .jsonc)
   ```

2. **Проверьте настройку порта:**
   ```bash
   # Порт должен совпадать
   echo $OPENCODE_PORT  # Обычно 4096
   opencode --port 4096 # Должен совпадать
   ```

3. **Убедитесь, что OpenCode запущен внутри tmux:**
   ```bash
   echo $TMUX  # Должно быть непустое значение
   ```

4. **Проверьте логи OpenCode:**
   ```bash
   tail -f ~/.config/opencode/logs/opencode.log
   ```

### Ghost pane и осиротевшие процессы

**Проблема:** tmux-pane остаются открытыми после завершения задач или накапливаются процессы `opencode attach`

**Это исправлено в последней версии.** Теперь жизненный цикл сессии корректно закрывает pane и завершает процессы.

**Как проверить, что фикс работает:**
```bash
# После запуска фоновых задач проверьте осиротевшие процессы
ps aux | grep "opencode attach" | grep -v grep
# Должно быть пусто

# Проверьте активные tmux-pane
tmux list-panes
# Должны остаться только pane основной сессии
```

**Если осиротевшие процессы все еще есть:**
1. **Завершите все осиротевшие процессы:**
   ```bash
   pkill -f "opencode attach"
   ```

2. **Закройте ghost pane:**
   ```bash
   # В tmux закрыть pane вручную
   tmux kill-pane -t <pane-id>
   ```

3. **Перезапустите OpenCode** с обновленным плагином

**Технические детали:**
Фикс реализует корректный lifecycle сессий:
- `session.abort()` вызывается после завершения задачи
- Graceful shutdown с Ctrl+C перед `kill-pane`
- Обработчики событий `session.deleted`
- Автоматическая очистка tmux-pane и процессов

Подробности реализации: [AGENTS.md](../AGENTS.md).

### Конфликты портов

**Проблема:** "Port already in use" или агенты не подключаются

**Решения:**
1. **Используйте другой порт:**
   ```bash
   export OPENCODE_PORT=5000
   opencode --port 5000
   ```

2. **Завершите существующие процессы OpenCode:**
   ```bash
   pkill -f "opencode"
   ```

3. **Проверьте конфликтующие сервисы:**
   ```bash
   netstat -tulpn | grep 4096
   ```

### Проблемы с tmux-сессиями

**Проблема:** не удается создать или подключить tmux-сессию

**Решения:**
1. **Установите tmux:**
   ```bash
   # Ubuntu/Debian
   sudo apt install tmux

   # macOS
   brew install tmux

   # Или используйте пакетный менеджер вашего дистрибутива
   ```

2. **Проверьте версию tmux:**
   ```bash
   tmux -V  # Должна быть 1.8 или выше
   ```

3. **Сбросьте конфигурацию tmux:**
   ```bash
   rm -f ~/.tmux.conf
   tmux kill-server
   tmux
   ```

### Проблемы с layout

**Проблема:** pane расположены не так, как ожидается

**Решения:**
1. **Попробуйте другой layout:**
   ```json
   {
     "tmux": {
       "enabled": true,
       "layout": "tiled",
       "main_pane_size": 40
     }
   }
   ```

2. **Отрегулируйте layout вручную:**
   ```bash
   # В tmux меняйте размер pane
   Ctrl+B Alt+Arrow Keys
   ```

3. **Очистите pane и перезапустите:**
   ```bash
   tmux kill-pane -a
   # Перезапустите OpenCode
   ```

### Проблемы производительности

**Проблема:** слишком много pane или низкая производительность

**Решения:**
1. **Уменьшите размер главной pane:**
   ```json
   {
     "tmux": {
       "enabled": true,
       "layout": "main-vertical",
       "main_pane_size": 40
     }
   }
   ```

2. **Сократите количество фоновых задач:**
   ```bash
   # В OpenCode используйте меньше параллельных операций
   # Или настройте агентов на более последовательную работу
   ```

3. **Очистите старые pane:**
   ```bash
   # Закрыть все pane, кроме текущей
   tmux kill-pane -a
   ```

## Расширенное использование

### Кастомная конфигурация tmux

Создайте `~/.tmux.conf` для тонкой настройки:

```bash
# Включить поддержку мыши
set -g mouse on

# Кастомные бинды
bind-key r source-file ~/.tmux.conf

# Улучшенные цвета
set -g default-terminal "screen-256color"

# Настройка status bar
set -g status-right "#H %Y-%m-%d %H:%M"

# Навигация по pane
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R
```

### Мониторинг нескольких агентов

Следите за конкретными агентами более детально:

```bash
# Мониторинг всех pane агентов
watch -n 1 'tmux list-panes -a -F "#{pane_current_command} (#{pane_index})"'

# Вывод только pane OpenCode
tmux list-panes -a | grep opencode

# Мониторинг фоновых задач
tmux list-panes -a | grep background
```

### Интеграция с фоновыми задачами

Плагин предоставляет инструменты фоновых задач, которые бесшовно работают с tmux:

| Инструмент | Описание | Интеграция с tmux |
|------------|----------|-------------------|
| `background_task` | Асинхронный запуск агентов | Создает pane для мониторинга |
| `background_output` | Проверка результатов задачи | Вывод появляется в pane |
| `background_cancel` | Остановка запущенных задач | Очищает pane |

### Скрипты и автоматизация

```bash
#!/bin/bash
# Скрипт автозапуска OpenCode с tmux

# Создать выделенную сессию
tmux new -d -s opencode

# Запустить OpenCode в этой сессии
tmux send-keys -t opencode:0 'opencode --port 4096' Enter

# Подождать и подключиться
sleep 2
tmux attach -t opencode
```

### Мониторинг логов

```bash
# Мониторинг логов OpenCode в реальном времени
tmux split-window -h
tmux send-keys 'tail -f ~/.config/opencode/logs/opencode.log' Enter

# Вернуться к основной pane
tmux select-pane -L
```

### Кастомные layout

Создавайте собственные раскладки pane:

```bash
# В tmux создать layout из 3 pane
tmux split-window -h
tmux split-window -v
tmux select-pane -L

# Сохранить layout
tmux save-buffer ~/my-layout.txt

# Восстановить позже
tmux load-buffer ~/my-layout.txt
```

## Лучшие практики

### Управление сессиями
- Используйте именованные сессии для разных проектов
- Отсоединяйтесь, когда активный мониторинг не нужен
- Периодически очищайте неиспользуемые сессии

### Производительность
- Используйте `main-vertical` или `main-horizontal` для лучшего фокуса
- Подбирайте `main_pane_size` под разрешение экрана
- Ограничивайте параллельные операции агентов в сложных задачах

### Рабочий процесс разработки
1. Запустите tmux-сессию для проекта
2. Запустите OpenCode с включенной интеграцией tmux
3. Наблюдайте за активностью агентов
4. Отсоединяйтесь при необходимости и подключайтесь снова для проверки прогресса
5. После завершения работы очищайте pane

### Отладка
- Используйте `Ctrl+B z`, чтобы увеличить конкретную pane
- Проверяйте логи, если агенты не отвечают
- Проверяйте настройки порта при переключении между проектами
